  • Обработка персональных данных — любое действие с персональными данными, совершаемое с использованием средств автоматизации или без их использования.
  • Оператор — ИП Лут Виталий Андреевич, самостоятельно организующий обработку персональных данных на сайте.
  • Субъект персональных данных — физическое лицо, посетитель сайта или лицо, направившее обращение через сайт.
  • Автоматизированная обработка — обработка персональных данных с помощью средств вычислительной техники.
  • Cookie — небольшой фрагмент данных, который сайт может сохранять в браузере посетителя для технических целей или (с согласия) для аналитики.
  • Посетитель сайта — любое лицо, обратившееся к сайту через сеть интернет.

03Какие данные могут обрабатываться

Сайт собирает персональные данные только тогда, когда посетитель сам добровольно направляет их через одну из форм сайта. В зависимости от формы могут обрабатываться следующие категории:

  • имя;
  • номер телефона;
  • контакт для связи, указанный пользователем самостоятельно (телефон, никнейм в мессенджере или иной способ связи — форма брифа не разделяет это на отдельные поля);
  • выбранный тип сайта и другие ответы на вопросы брифа (стадия проекта, цель, имеющиеся материалы, желаемые сроки, предпочитаемый канал связи);
  • комментарий, добровольно оставленный пользователем в свободной форме.

Сайт не запрашивает и не собирает через формы адрес электронной почты как отдельное обязательное поле, бюджет проекта, файлы или вложения — на момент публикации настоящей Политики такие поля в формах отсутствуют.

Дополнительно, в связи с работой сайта и (при наличии согласия) аналитикой, могут обрабатываться технические данные:

  • IP-адрес (в хэшированном виде — см. раздел 8);
  • сведения о браузере и операционной системе (user-agent);
  • тип устройства;
  • адрес страницы, с которой отправлено обращение;
  • источник перехода и UTM-метки;
  • дата и время обращения;
  • — после согласия на аналитические cookie — действия на сайте, посещённые страницы, карта кликов и аналогичные данные, которые собирает Яндекс.Метрика.

Сайт не собирает и не обрабатывает паспортные данные, данные банковских карт, биометрические персональные данные и специальные категории персональных данных.

05Правовые основания обработки

В зависимости от ситуации обработка персональных данных основывается на:

  • согласии субъекта персональных данных;
  • необходимости совершения действий по инициативе субъекта персональных данных до заключения договора (обработка заявки, обсуждение проекта, подготовка предложения);
  • необходимости заключения и исполнения договора, стороной которого является субъект персональных данных;
  • необходимости исполнения обязанностей, предусмотренных законодательством Российской Федерации.

Не любое из перечисленных оснований применяется к любому виду обработки одновременно — конкретное основание зависит от того, на каком этапе взаимодействия находится пользователь.

06Способы обработки

Обработка персональных данных осуществляется с использованием средств автоматизации (формы сайта, электронная почта, мессенджеры) и, при необходимости, без их использования. Оператор совершает следующие действия с персональными данными: сбор, запись, систематизация, накопление, хранение, уточнение, использование, передача (предоставление, доступ) уполномоченным техническим сервисам в необходимом объёме, блокирование, удаление, уничтожение.

07Сроки обработки и хранения

  • персональные данные обрабатываются до достижения цели обработки, ради которой они были предоставлены;
  • если обработка основана на согласии — до отзыва согласия субъектом персональных данных;
  • если между сторонами заключён договор — в течение сроков, необходимых для его исполнения, и сроков, установленных законодательством для хранения соответствующей документации;
  • технические данные, обрабатываемые сервисом Яндекс.Метрика, хранятся в соответствии с настройками и правилами этого сервиса.

Оператор не устанавливает бессрочное хранение персональных данных и удаляет данные по достижении цели их обработки либо по обращению субъекта персональных данных.

08Передача третьим лицам и используемые сервисы

Оператор не продаёт и не передаёт персональные данные третьим лицам в целях, не связанных с работой сайта и обработкой обращений. Для функционирования сайта и обработки заявок используются следующие технические сервисы:

  • GitHub Pages — размещение и доставка сайта пользователю. Как и любой хостинг-провайдер, GitHub Pages обрабатывает стандартные технические данные HTTP-запросов (IP-адрес, заголовки запроса) в объёме, необходимом для доставки страниц.
  • Supabase — инфраструктура для приёма и хранения заявок и брифов, отправленных через формы сайта. Данные, введённые пользователем в форму (имя, контакт, ответы на вопросы брифа, комментарий), передаются в Supabase и сохраняются там для последующей обработки оператором. IP-адрес передаётся и хранится в хэшированном виде и используется только для защиты форм от автоматических злоупотреблений.
  • Telegram и MAX — оператор получает уведомление о новой заявке или брифе в мессенджер (Telegram и/или MAX) в виде сообщения, содержащего указанные пользователем имя, контакт и ответы на вопросы формы. Это сделано для оперативной обработки обращений и является частью фактической реализации сайта.
  • Яндекс.Метрика — веб-аналитика, запускается только после согласия пользователя на аналитические технологии (см. раздел 9).

Передача данных указанным сервисам ограничена объёмом, необходимым для достижения соответствующей цели, и не включает передачу данных для целей, не связанных с работой сайта. Google Analytics, Meta Pixel и иные сервисы, не перечисленные выше, на сайте не используются.

Важно. Оператор не обещает, что персональные данные хранятся исключительно на серверах, принадлежащих оператору. Часть данных обрабатывается с использованием инфраструктуры перечисленных выше сервисов, что является обычной практикой для сайтов такого рода.

09Яндекс.Метрика

На сайте используется сервис веб-аналитики Яндекс.Метрика, счётчик № 104469602. Сервис используется для:

  • аналитики посещаемости сайта;
  • карты кликов;
  • отслеживания переходов по ссылкам;
  • точного показателя отказов;
  • Вебвизора (записи действий посетителей на сайте для анализа удобства интерфейса).

Яндекс.Метрика запускается только после того, как пользователь выбрал «Принять» в баннере cookie на сайте. До этого выбора запросы к Яндекс.Метрике не отправляются. Пользователь может отказаться от аналитики, выбрав «Только необходимые» в баннере cookie, или изменить свой выбор позже на странице Cookies.
